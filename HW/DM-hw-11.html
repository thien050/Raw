Scaling, fitting logistic regression model

The 6 highest association by predictor variables based on previous data prep was used as predictor variables.

telecom[,c("OnlineSecurity", "TechSupport", "InternetService", "PaymentMethod", 
                         "OnlineBackup", "DeviceProtection")] <- scale(telecom[,c("OnlineSecurity", "TechSupport", "InternetService", "PaymentMethod", 
                         "OnlineBackup", "DeviceProtection")])
                         
glm_telecom <- glm(
  formula = target ~ OnlineSecurity + TechSupport + InternetService + PaymentMethod + OnlineBackup +         
    DeviceProtection,  
  family = binomial(link = "logit"),
  data = telecom %>%
    dplyr::filter(partition == "Train") 
)

From the summary output, all predictor variables in this model is significant with p value < alpha = 0.5. The variable with the biggest impact based on the z statistics is Online Security with a z value of 10.22. A coefficient of 0.4710 means that the odds of churning is 1.5 times that of not churning when a customer has online security, keeping all other variables constant. This may sound reversed but it could mean that their online security system is faulty which could lead to customers churning or leaving the service.

Comparing prediction accuracy

telecom <- telecom %>%
  dplyr::mutate(
    predict_probability = predict(
      object = glm_telecom,
      newdata = telecom,
      type = "response"  
    ),
    predict_class = round(predict_probability)  
  )

prediction_Train <- ROCR::prediction(
  predictions = telecom$predict_probability[telecom$partition == "Train"],
  labels = telecom$target[telecom$partition == "Train"]
)
ROCR::performance(
  prediction.obj = prediction_Train,
  measure = "auc"
)@y.values
## [[1]]
## [1] 0.79

For the training set the area under curve is .79.

prediction_Test <- ROCR::prediction(
  predictions = telecom$predict_probability[telecom$partition == "Test"],
  labels = telecom$target[telecom$partition == "Test"]
)
ROCR::performance(
  prediction.obj = prediction_Test,
  measure = "auc"
)@y.values
## [[1]]
## [1] 0.79

At .79 for the test data, our model does a decent job discriminating on the test set as well. The tightness indicates good generalization.
